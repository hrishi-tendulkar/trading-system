Strategy Interpretation

What edge this sleeve is trying to capture

Breakout Confirmation is the cleanest continuation sleeve because it waits for proof through price instead of assuming that leadership will continue on hope alone.

Use it when trend structure is intact, sector context is supportive, and a definable trigger exists. A good signal is close enough to the trigger that invalidation remains obvious. A false positive is a watch candidate being treated as if the breakout already happened.

Weekly Reading Guide

How to use this page in the loop

Open this page from overview when a promoted name cites `Breakout Confirmation` as its source or when you need to decide whether a `Wait for confirmation` row deserves extra attention this week.

The page should help you distinguish three different truths quickly: live triggered action, valid watch-only interest, and technically attractive names that risk rules still suppress.

Canonical Rules

Plain-language rule spine first

1

Entry conditions

Trend intact, resistance clearly defined, and breakout must actually trigger before the page upgrades the action.

2

Context requirements

Relative strength should be acceptable or strong, and sector context should confirm rather than fight the move.

3

Exclusions and suppressors

Defensive-regime weakness, poor sector confirmation, and event-freeze risk rules can block promotion even if the chart looks attractive.

4

Holding style

Short-term continuation sleeve targeting roughly `1` to `15` trading days with proof-first timing.

5

Invalidation or reassess

Failed breakout back into range or immediate loss of sector support should force a fast reassess.

6

Board eligibility

`Buy now` and `Wait for confirmation` can feed the board. `No action` and suppressed rows stay off the fresh-cash action board.

Exact field logic

`Trend intact` + `definable breakout trigger` + `relative strength acceptable or strong` + `supportive sector context` + `triggered entries only` + `risk-rule suppression can override board promotion`.
