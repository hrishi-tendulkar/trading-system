# Prototype Reference

## Purpose

The prototype exists to show the current thinking around workflow, page types, and data relationships.

It should not be copied as the final design.

Use it to understand:

- weekly summary is the main hub,
- full candidate board explains what passed and failed,
- stock detail is a reusable page for each stock,
- strategy detail is a reusable page for each strategy,
- archive should preserve old weekly runs,
- pages should connect to each other clearly.

## How To Interpret The Prototype

The prototype is useful for:

- seeing the main user flow,
- seeing example data fields,
- understanding how weekly summary, board, stock detail, and strategy detail relate,
- identifying UX problems to improve.

The prototype is not useful as:

- final visual design,
- final navigation model,
- final wording,
- final layout,
- final component system.

We want a better UX, not a prettier version of this prototype.
