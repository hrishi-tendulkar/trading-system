# MEMORY_DESIGN

## Durable Notes

- [2026-05-22] Interfaces should optimize for fast weekly scanning under limited attention, not for maximal information density.
- [2026-05-22] User-facing pages should not spend prime screen space explaining the product to the operator; the top of the weekly view should start with market posture, cash deployment, and stock decisions, with glossary/help content available on demand.
- [2026-05-24] The scalable UI rule is stable page identity with deeper modules underneath it: weekly remains the decision session, daily remains the exception layer, and deep dive expands through richer evidence instead of new top-level screens.
- [2026-05-25] Strategy pages should encode three separate truths in the UI: strategy trust level, workflow promotion status, and row-level live candidate state. If those collapse into one badge or one color, the page overstates confidence.
