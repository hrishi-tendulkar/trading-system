# CONTEXT_CURATION

## Decisions

## [2026-05-22] Keep curated context slim and durable
**Decision** · Source: external blueprint adopted into this repo
Context files should store decisions, patterns, gotchas, and don'ts, not task lists or long historical logs.
**Why it matters:** Context only works when agents can load it every session without drowning in stale detail.

## Patterns

## [2026-05-22] The system decays when maintenance depends on memory alone
**Pattern** · Source: external blueprint adopted into this repo
If context upkeep is not made explicit and routine, it quickly drifts away from actual project reality.
**Why it matters:** The curator role and weekly review are structural requirements, not optional process extras.

## Don'ts

## [2026-05-22] Do not auto-edit north-star docs without explicit approval
**Don't** · Source: external blueprint adopted into this repo
Foundational domain docs should only change through an intentional strategic review, not routine maintenance.
**Why it matters:** This protects project identity from low-signal churn and accidental contradiction.
